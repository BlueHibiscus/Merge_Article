Understanding DML Commands Of SQL Server With Examples

What is SQL?

Structured Query Language (SQL) is a programming language developed in the 1970s by IBM researchers. It enables creation and manipulation of databases and uses certain commands to carry out the operations designed to facilitate the interaction between databases. SQL statements are classified in various ways which are grouped into sublanguages. The term “Sublanguage” was first brought out by E.F. Codd in 1970 that refers to a computer language used to define or manipulate the structure and contents of a Relational database management system (RDBMS). Sublanguages in SQL is a fascinating technology. Once you get a hang of it, you will no doubt see its value. I, however, in this brief article am specifically picking the DDL aspect of the Sub-language, hope this makes your understanding easier with the examples used.

MERGE:

MERGE statement was introduced in 2008 to modify data in the target table based on data from the source table. Users can then insert, modify, or delete data all in one statement according to how the rows match up as a result of the join.

Syntax : Merge into [ target_table] using [ source_table] on [ condition]

WHEN MATCHED THEN [ update command ]

WHEN MATCHED THEN [ delete command ]

WHEN NOT MATCHED THEN [ insert command]

Ex: To merge the records from New_table into Old_table using the Insert command.

Step1: CREATE OLD_TABLE (Target table) and NEW_TABLE (Source table) and INSERT records.

Ex: Write the Merge command to Update records in OLD_TABLE and Insert Records from NEW_TABLE into the OLD_TABLE.

Step2: The first MERGE clause is WHEN MATCHED. This clause is used when you want to update or delete rows in the target table that match rows in the source table. With matching EMPID the OLD_TABLE’S ENAME and SALARY columns are updated with NEW_TABLE columns.

Step3: To write the Merge command to Insert records into the OLD_TABLE from the NEW_TABLE, using WHEN NOT MATCHED clause.

Step4: Using Merge command to Delete records in the OLD_TABLE from the NEW_TABLE. We are using WHEN MATCHED clause here again.

Step5: Using Merge command to Insert, Delete and Update records from the NEW_TABLE(Source table) into the OLD_TABLE(Target table), using WHEN MATCHED and WHEN NOT MATCHED Clause.

Step 6: Detailed report of Data Manipulations performed on every record in the OLD_TABLE and NEW_TABLE (Target and Source tables) through Merge statements.

Notes:

1) Every MERGE statement must end with a semi-colon.

2) SELECT @@RowCount after writing the MERGE statement which will return the number of records that have been modified by the transactions.

SUMMARY:

The example of the Merge statement used in this article helps to perform Data Manipulations using INSERT, DELETE and UPDATE commands all in one statement or otherwise. The Source and Target tables combined with WHEN MATCHED and WHEN NOT MATCHED clauses perform necessary operations making the flow of complex queries easier to understand and perform.
